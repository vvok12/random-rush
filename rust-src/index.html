Use /playroom-ws endpoint to connect to web socket

 

On current API assumptions


The user is an active user. In the future, it is supposed to be authenticated with a permanent account. For now, it is created for every WebSocket connection.

The playroom is where the game happens. In the future, our users may be able to connect to certain playrooms. For now, the server will return a hard coded ID.

Each playroom has a single board. Any user can ask for board status at any time.
